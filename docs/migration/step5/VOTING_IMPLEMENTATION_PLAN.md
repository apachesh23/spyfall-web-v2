# План внедрения голосования (Colyseus, без Supabase)

Документ фиксирует целевую архитектуру и этапы. Логика авторитетна на **game-server** (`SpyfallRoom`); клиент только отображает `GameState` и шлёт сообщения.

## 1. Контракт состояния (`GameState`)

Поля (синхронизировать `packages/shared` и `apps/game-server/src/schema`):

| Поле | Назначение |
|------|------------|
| `phase` | `discussion` \| `voting` |
| `gameStartedAt` | `Date.now()` при создании комнаты |
| `votingDurationSec` | Длительность фазы «отдать голос» (из настроек лобби `vote_duration`, мин → сек) |
| `firstEarlyVoteAfterAt` | Не раньше этого времени можно собрать досрочное (прод: +3 мин от старта; **dev: 10 с**) |
| `earlyVoteCooldownUntil` | После завершения **досрочной** сессии голосования — не раньше этого снова досрочное (прод: +3 мин; **dev: 10 с**) |
| `earlyVotesUsed` | Сколько досрочных голосований уже было (макс. 2 за партию) |
| `voteStage` | Подфаза голосования (см. ниже) |
| `voteEndsAt` | Дедлайн текущего раунда сбора голосов (0 вне сбора) |
| `voteTransitionEndsAt` | Конец промежуточного экрана (5 с), 0 если не в переходе |
| `revoteA`, `revoteB` | Идентификаторы двух кандидатов во 2-м раунде |
| `stubEliminatedId` | Временный UI: «выгнали игрока» (пока без реального game over) |
| `earlyVoteAck` | `Map<playerId, "1">` — кто нажал «досрочно» (toggle) |
| `voteBallots` | `Map<voterId, "skip" \| targetPlayerId>` |

### Значения `voteStage`

- `idle` — идёт обсуждение (при `phase === "discussion"`).
- `collect1` — первый раунд голосования.
- `intermission_no_vote` — «Голосование не состоялось», 5 с → `discussion`.
- `intermission_tie` — двое в ничьей, 5 с → `collect2`.
- `collect2` — повторное голосование только между `revoteA` и `revoteB`.
- `intermission_revote_no_vote` — после 2-го раунда ничья/все пропустили, 5 с → `discussion`.
- `stub_eliminated` — тестовый экран исключения, 5 с → `discussion`.

## 2. Сообщения клиент → сервер

Расширить `WS_CLIENT_MESSAGE` в `packages/shared` и дубле `matchContract.ts`:

- `earlyVoteToggle` — переключить своё участие в досрочном (>50% при других условиях запускает голосование).
- `voteCast` — тело `{ targetId: string }` (раунд 1: любой другой игрок; раунд 2: только `revoteA` или `revoteB`).
- `voteSkip` — засчитать «пропуск» как ответившего (для условия «все ответили»).

## 3. Правила (сводка)

### Досрочное голосование

- Порог: **строго больше 50%** игроков → `Math.floor(n/2) + 1` нажатий.
- Не более **2** досрочных за партию.
- Первое досрочное: не раньше **3 мин** после старта (в коде пометка + **dev 10 с**).
- Пауза между досрочными: **3 мин** после **закрытия** предыдущей досрочной сессии (**dev 10 с**).
- Старт по **таймеру обсуждения** не тратит лимит досрочных и не двигает cooldown.

### Сбор голосов (раунд 1 и 2)

- Завершение сбора: **все живые (в р2 — все не-кандидаты) ответили** ИЛИ `voteEndsAt`.
- Голос можно **менять**, пока идёт сбор.
- «Пропуск» не даёт голоса кандидату; если **все** ответили **skip** → голосование не состоялось.
- Подсчёт: только бюллетени с валидным `targetId`; при нуле голосов за игроков → не состоялось.
- Ничья **ровно у двух** лидеров → экран перехода → раунд 2.
- Ничья **тройная и больше** → не состоялось (упрощение).

### Раунд 2

- Голосуют только игроки **кроме** двух кандидатов; голоса кандидатов игнорируются.
- Учитываются только голоса за `revoteA` или `revoteB`.
- Равные голоса или отсутствие голосов за кандидатов → не состоялось.

### Промежуточные экраны

- 5 с обратного отсчёта внизу сплэша (`resultCountdown` в UI), сервер переводит по `voteTransitionEndsAt`.

## 4. Клиент (web)

- Перенести **только UI** из `docs/migration/step5/voting-components/` в `apps/web/src/features/match/components/voting/`, подправить импорты (`LottieIcon`, `PlayerAvatar`, тип игрока из матча).
- Новый контейнер (например `MatchVotingOverlay`) читает срез из `GameStateJson`, шлёт `room.send`.
- `MatchGamePageLayout`: досрочный блок — счётчик нажатий, toggle, блокировка по времени, лейбл `ГОЛОСОВАТЬ x/y`.
- `VotingSplash`: для фазы сбора **не** вызывать `onClose` по нулю таймера — переходы только с сервера.

## 5. Старт матча

- В `buildMatchRoomOptions` и `POST /api/game/start` передавать `votingDurationMs` из `settings.vote_duration` (минуты → мс, с разумными clamp).

## 6. Поведение после голосования

Если к моменту возврата в `discussion` основной таймер обсуждения уже истёк, сервер продлевает дедлайн на **2 минуты** (`DISCUSSION_EXTENSION_AFTER_VOTE_MS` в `SpyfallRoom`), иначе следующий тик снова мгновенно запускает голосование.

## 7. Дальнейшие шаги (не в этом этапе)

- Реальные сплэши исходов, удаление игрока из ростра, финал шпион/мирные.
- Пауза матча во время голосования (при необходимости сдвиг `voteEndsAt`).

## 8. Чеклист тестов (ручной)

- Досрочное: toggle, снятие голоса, порог >50%, лимит 2, таймер первого старта и cooldown (dev 10 с).
- Старт по окончанию обсуждения без досрочного.
- Все проголосовали до дедлайна; часть — по дедлайну; все skip; победитель; ничья двоих → раунд 2; ничья в раунде 2.
