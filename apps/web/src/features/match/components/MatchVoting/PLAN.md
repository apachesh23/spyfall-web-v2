# MatchVoting — план UI-архитектуры голосования

Цель: один **глобальный** этап `VOTE` на уровне матча, внутри которого меняются только подфазы и куски UI. Сейчас реализуется **только оболочка и композиция**; привязка к Colyseus/стейт-машине — позже.

## Слои

1. **`MatchVoteRoot`**  
   Корневой full-screen оверлей: фон, слот контента по центру, звуки входа/выхода, таймерная логика, CSS-переменные цветов. Стили корня — `MatchVoteRoot.module.css`.  
   Пока корень смонтирован, **не размонтируем** нижний HUD ради смены мини-фазы — обновляются только текст и число.

2. **`VotingHeadlineStrips`** (условное «Title»)  
   Верхняя медиа-композиция: две полоски, бегущие строки «раунд» и «подпись голосования», картинка полосы (`vote_stipe` / `final_stipe`), общая тень `shadow_stipe`.  
   Проп **`visible`** + `AnimatePresence`: при смене подфазы внутри того же корня полоски можно **свернуть** и снова **показать** отдельной анимацией, не закрывая весь оверлей.

3. **`VotingSubtitleStrip`** (нижний HUD)  
   Короткая подпись + таймер на полосках. Вход/выход синхронизирован с **жизненным циклом корня** (mount/unmount `MatchVoteRoot`). Внутри голосования — только смена `label` / `value` без пересборки дерева полосок.

4. **Мини-фазы (контент)**  
   Отдельные компоненты под сценарии центра:
   - `VotingPhaseRevoteCandidates` — «кандидаты на повторное голосование»;
   - `VotingPhaseNoVote` — «голосование не состоялось».  
   Корень остаётся на экране, в центре подменяется `children` или явный `centerPhase`.

## Конфиг

**`voting.config.ts`** — единый файл: геометрия декора, тайминги полосок, HUD, пути к ассетам и текстовые пресеты.

В корне `MatchVoting/` лежат только **`index.ts`**, **`voting.config.ts`**, **`matchVoteRoot.types.ts`**, **`PLAN.md`**. Каждый UI-блок — подпапка `ИмяКомпонента/` с `ИмяКомпонента.tsx`, при необходимости `.module.css` и **`index.ts`** (реэкспорт для барреля).

## Именование

| Идея пользователя | Компонент в коде        |
|-------------------|-------------------------|
| Title / шапка     | `VotingHeadlineStrips`  |
| Subtitle / низ    | `VotingSubtitleStrip`   |

## Интеграция с матчем

**`MatchVotingOverlay`** — адаптер Colyseus / `voteStage` → пропсы `MatchVoteRoot`, отправка `voteCast` / `voteSkip`. Классы списка и кнопок — `MatchVotingOverlay/MatchVotingOverlay.module.css` (реэкспорт как `votingSplashStyles` в корневом `index.ts` для дев-панели).

## Дев-панель (лобби)

Кнопки сценариев открывают `MatchVoteRoot` с разными комбинациями: раунд 1/2, обычное/финальное голосование, скрытые верхние полоски, мини-фазы «повтор» / «не состоялось».
